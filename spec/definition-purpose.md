# 프로젝트 정의 및 목적

## 1. 프로젝트 개요

### 1.1 프로젝트명
**개발자 온보딩 애플리케이션 (Developer Onboarding Application)**

### 1.2 프로젝트 정의

신규 입사자가 팀에 빠르게 적응하고 개발 환경을 구축하며, 팀의 개발 프로세스와 문화를 이해할 수 있도록 돕는 **오프라인 기반 데스크톱 애플리케이션**입니다.

웹 개발자가 친숙한 **React** 기반으로 개발되며, **Electron**을 통해 인터넷 연결 없이 독립적으로 실행 가능한 실행 파일로 배포됩니다.

---

## 2. 프로젝트 목적

### 2.1 주요 목적

| 목적 | 설명 |
|------|------|
| **일관된 온보딩** | 모든 신규 개발자에게 동일한 품질의 온보딩 경험 제공 |
| **환경 구축 자동화** | 개발 환경 세팅 과정을 체계화하여 설치 시간 단축 |
| **지식 전달 효율화** | 팀의 개발 프로세스, 코드 구조, 인프라 등을 체계적으로 전달 |
| **오프라인 접근성** | 인터넷 연결 없이도 언제든지 학습 가능 |
| **진행 상황 추적** | 학습 진도를 시각화하여 동기 부여 및 관리 용이성 제공 |

### 2.2 해결하고자 하는 문제

#### 기존 문제점
- 온보딩 과정이 구두 전달에 의존하여 정보 누락 발생
- 개발 환경 세팅 문서가 분산되어 있어 찾기 어려움
- 신규 개발자별로 온보딩 품질 편차 존재
- 온보딩 진행 상황을 파악하기 어려움

#### 개선 효과
- 체계화된 문서를 통한 정보 전달 일관성 확보
- 원스톱 환경에서 모든 온보딩 콘텐츠 접근
- 진도 추적 기능을 통한 학습 동기 부여
- 재사용 가능한 온보딩 템플릿 확보

## 3. 타겟 사용자

### 3.1 주요 사용자

| 사용자 | 특징 | 요구사항 |
|--------|------|----------|
| **신규 백엔드 개발자** | Python/Docker 환경 필요 | 백엔드 환경 세팅, API 구조 이해 |
| **신규 프론트엔드 개발자** | React/Next.js 환경 필요 | 프론트 환경 세팅, 컴포넌트 구조 이해 |
| **경력 개발자** | 빠른 적응 필요 | 핵심 내용 위주의 빠른 스캔 가능 |

### 3.2 사용자가 앱을 통해 얻는 것

1. **계정 체크리스트** - 필수 계정 초대 확인 및 접근 권한 획득
2. **개발 환경 세팅** - 로컬/스테이징/프로덕션 환경 구성 방법 습득
3. **개발 프로세스 이해** - 기획부터 배포까지 전체 파이프라인 이해
4. **커뮤니케이션 방법** - 팀 내 문서화, 브랜치 전략 학습
5. **코드/인프라 파악** - 핵심 비즈니스 로직, 클라우드 인프라 이해
6. **운영 프로세스** - CI/CD, 모니터링, CS 지원 방법 습득

## 4. 핵심 기능

### 4.1 콘텐츠 구조

| 파트 | 주요 내용 |
|------|----------|
| **Part 1** | 계정 체크 (GitHub, Notion, Discord, Google Calendar) |
| **Part 2** | 세팅 환경 설명 (Docker, Python, Frontend, 배포 환경) |
| **Part 3** | 개발 프로세스 (기획~배포 전체 과정) |
| **Part 4** | 개발 커뮤니케이션/문서 (API 문서, 브랜치 전략) |
| **Part 5** | 코드 - 어플리케이션단 (비즈니스 로직, 결제/플랜) |
| **Part 6** | 클라우드 (사용 중인 서비스, 학습 필요 항목) |
| **Part 7** | CI/CD |
| **Part 8** | 모니터링 및 데이터 분석 (CS 지원, 로깅) |
| **Part 9** | 이후 해결해야 할 문제/고민할 것 |

### 4.2 인터랙티브 기능

- **체크리스트 컴포넌트** - 각 항목 완료 시 체크하여 진도 표시
- **진행률 표시** - 전체 및 각 파트별 완료율 시각화
- **사이드바 내비게이션** - 전체 목차와 파트별 빠른 이동
- **로컬 저장소** - 모든 진행 상황을 LocalStorage에 저장하여 세션 유지

## 5. 기술적 특징

### 5.1 기술 스택 선정 이유

| 기술 | 선정 이유 |
|------|----------|
| **React** | 웹 개발자가 친숙한 UI 프레임워크로, 유지보수 용이 |
| **Electron** | 오프라인 실행 가능한 독립형 실행 파일 배포 |
| **zustand** | 가볍고 간단한 상태 관리, LocalStorage 연동 용이 |
| **HeroUI** | React Aria 기반의 접근 가능한 UI 컴포넌트 라이브러리 |
| **MDX** - 마크다운에 React 컴포넌트를 임베드하여 풍부한 콘텐츠 작성 가능 |

### 5.2 오프라인 우선 설계

- **외부 통신 없음** - 모든 데이터가 로컬에 저장
- **독립 실행** - 인터넷 연결 상관없이 언제든지 학습 가능
- **실행 파일 배포** - Electron을 통해 .exe/.dmg 등으로 패키징

## 6. 프로젝트 범위

### 6.1 포함 (In Scope)

- 9개 파트로 구성된 온보딩 콘텐츠 제공
- 체크리스트 및 진행률 추적 기능
- MDX 기반 콘텐츠 렌더링 시스템
- 오프라인 데스크톱 애플리케이션
- LocalStorage 기반 데이터 영속성

### 6.2 미포함 (Out of Scope)

- 실제 계정 초대 발송 기능
- 개발 환경 자동 설치 스크립트
- 서버와의 데이터 동기화
- 다중 사용자 지원
- 웹 배포 (Electron 데스크톱 앱 전용)
